# ⭐ What Is `poll()`?

`poll()` is a system call that allows you to monitor **multiple file descriptors** at once:

* sockets
* stdin
* pipes
* files (less useful here)

It lets you ask:

> “Tell me when something happens on ANY of these sockets.”

In an IRC server, this is how you:

* detect **new clients** connecting to your listening socket
* detect **messages** from existing clients
* detect when a client disconnects

This avoids creating threads — everything happens in **one loop**.

---

# ⭐ The `struct pollfd`

This is the structure you pass to poll:

```c
struct pollfd {
    int   fd;        // File descriptor you're watching
    short events;    // What events we are waiting for
    short revents;   // What events actually occurred
};
```

### Let’s break it down:

### ✔ `fd`

The file descriptor you want to monitor.

Examples:

* The listening server socket
* A connected client’s socket

### ✔ `events`

What you want to check for:

Common flags:

* `POLLIN` → data to read
* `POLLOUT` → ready to write
* `POLLERR` → error
* `POLLHUP` → client hung up (disconnect)

For an IRC server you will almost always use:

```c
pfd.events = POLLIN;
```

### ✔ `revents`

After `poll()` returns, this field tells you **what actually happened**.

Example:

```c
if (pfd.revents & POLLIN) {
    // The fd has data to read
}
```

---

# ⭐ How `poll()` Works

Function signature:

```c
int poll(struct pollfd *fds, nfds_t nfds, int timeout);
```

Parameters:

* `fds`: an array of `pollfd`
* `nfds`: how many elements in the array
* `timeout`:

  * `-1` → wait forever
  * `0` → return immediately
  * `>0` → wait that many milliseconds

---

# ⭐ Example: Minimal Poll Loop

```c
struct pollfd fds[10];

fds[0].fd = server_socket;   // our listening socket
fds[0].events = POLLIN;      // wait for new connections

int result = poll(fds, 1, -1); // wait forever

if (result > 0)
{
    if (fds[0].revents & POLLIN)
    {
        // A new client is connecting!
        int client = accept(server_socket, NULL, NULL);
        printf("New client connected! fd=%d\n", client);
    }
}
```

---

# ⭐ Using Poll With Multiple Clients

After accepting a client, you add it to the list:

```c
fds[clientIndex].fd = client_socket;
fds[clientIndex].events = POLLIN;
```

Then on each loop:

```c
for (int i = 0; i < nfds; i++)
{
    if (fds[i].revents & POLLIN)
    {
        if (fds[i].fd == server_socket)
        {
            // new incoming connection
        }
        else
        {
            // data from a client
            recv(fds[i].fd, buffer, sizeof(buffer), 0);
        }
    }
}
```

# ⭐ Summary Table

| Field     | Meaning                                    |
| --------- | ------------------------------------------ |
| `fd`      | The socket we are monitoring               |
| `events`  | Which events we want (`POLLIN`, `POLLOUT`) |
| `revents` | What actually happened (set by poll)       |

# ⭐ Why `poll()` is Used in IRC Servers

You need to:

* handle maybe **100+ clients**
* with **one thread**
* without blocking

`poll()` makes this easy:

* It waits until **any client** has data
* It tells you which client(s) need processing
* You read/write only when ready

This is better than:

* `select()` (limited to 1024 fds)
* blocking `recv()` (would freeze server)
